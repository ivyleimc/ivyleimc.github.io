1. 新增的基本类型

JavaScript 有七种内置类型:

  • 空值(null)
  • 未定义(undefined)
  • 布尔值( boolean)
  • 数字(number)
  • 字符串(string)
  • 对象(object)
  • 符号(symbol，ES6 中新增)

除对象之外，其他统称为“基本类型”

typeof null === "object"; // true 历史问题
typeof undefined === "undefined"; // true
typeof true === "boolean"; // true
typeof 42 === "number"; // true
typeof "42" === "string"; // true
typeof { life: 42 }  === "object"; // true
typeof Symbol() === "symbol"; // true

2. 为什么要新加它

ES5 的对象属性名都是字符串，这容易造成属性名的冲突 若保证每个属性的名字都是独一无二的，就可从根本上防止属性名的冲突 这就是 ES6 引入 Symbol 的原因

于是 ES6 引入了一种新的原始数据类型 Symbol，表示独一无二的值

1
2

Symbol() === Symbol(); // false
Symbol("w") === Symbol("w"); // false

3.2 Symbol 函数前不加 new

Symbol函数前不能使用 new 命令，否则会报错 因为 Symbol 是一个原始类型的值，不是对象 也就是说，由于 Symbol 值不是对象，所以也不能添加属性 基本上，它是一种类似于字符串的数据类型

3.3 接受参数作为描述

Symbol 函数可以接受一个字符串作为参数，表示对 Symbol 实例的描述，主要是为了在控制台显示，或者转为字符串时，比较容易区分

s1.description // "foo"
s2.description // "bar"

s1.toString() // "Symbol(foo)"
s2.toString() // "Symbol(bar)"

4. 作为属性名的 Symbol

由于每一个 Symbol 值都是不相等的，这意味着 Symbol 值可以作为标识符，用于对象的属性名，就能保证不会出现同名的属性

let mySymbol = Symbol();

// 第一种写法
let a = {};
a[mySymbol] = 'Hello!';

// 第二种写法
let a = {
  [mySymbol]: 'Hello!'
};

// 第三种写法
let a = {};
Object.defineProperty(a, mySymbol, { value: 'Hello!' });

// 以上写法都得到同样结果
a[mySymbol] // "Hello!"

4.1 不能用点运算符

1
2
3
4
5

const mySymbol = Symbol();
const a = {};

a.mySymbol = 'Hello!';
a[mySymbol] // undefined
a['mySymbol'] // "Hello!"

点运算符后面总是字符串，所以不会读取 mySymbol 作为标识名所指代的那个值，导致a的属性名实际上是一个字符串，而不是一个 Symbol 值

同理，在对象的内部，使用 Symbol 值定义属性时，Symbol 值必须放在方括号之中

  }

// 123

4.2 遍历不到它

Symbol 作为属性名，遍历对象的时候，该属性不会出现在 for…in、for…of 循环中，也不会被 Object.keys()、Object.getOwnPropertyNames()、JSON.stringify() 返回。

但是，它也不是私有属性，有一个 Object.getOwnPropertySymbols()方法，可以获取指定对象的所有 Symbol 属性名。该方法返回一个数组，成员是当前对象的所有用作属性名的 Symbol 值

Object.getOwnPropertyDescriptors() 和 Object.getOwnPropertyDescriptor() 里能找到它

const obj = {};
let a = Symbol('a');
let b = Symbol('b');

obj[a] = 'Hello';
obj[b] = 'World';

const objectSymbols = Object.getOwnPropertySymbols(obj);

objectSymbols
// [Symbol(a), Symbol(b)]
