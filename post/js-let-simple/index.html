let 简介

All code test in Safari Version 12.1.1（14607.2.6.1.1）

1. 声明变量

ES6 新增了 let 命令，用来声明变量，用法类似于 var，但是所声明的变量，只在 let 命令所在的代码块内有效

1
2
3
4
5

if (true) {
  let a = 10;
  console.log(a);
}
// 10

2. 块作用域内有效

用法类似于 var，但是所声明的变量，只在 let 命令所在的代码块内有效

1
2
3
4
5

if (true) {
  let b = 10;
}
console.log(b);
// ReferenceError: Can't find variable: b

3. 不存在变量提升

1
2
3
4
5

if (true) {
  console.log(c);
  let c = 10;
}
// ReferenceError: Cannot access uninitialized variable.

块内若用 let 声明了一个变量， 那么在声明前就不能用它(typeof 也不行)，否侧会报 ReferenceError

4. 不能重复声明

1
2
3
4
5

if (true) {
  let d = 10;
  let d = 11;
}
// SyntaxError: Cannot declare a let variable twice: 'd'.

5. 适合用在 for 循环

1
2
3
4
5
6
7
8

var a = [];
for (var i = 0; i < 10; i++) {
  a[i] = function () {
    console.log(i);
  };
}
a[6]();
// 10

执行上面代码没有如愿输出6， 你应该懂什么问题吧 下面改用 let

1
2
3
4
5
6
7
8

var a = [];
for (let i = 0; i < 10; i++) {
  a[i] = function () {
    console.log(i);
  };
}
a[6]();
// 6

变量 i 是 let 声明的，当前的 i 只在本轮循环有效，所以每一次循环的 i 其实都是一个新的变量，所以最后输出的是6

你可能会问，如果每一轮循环的变量 i 都是重新声明的，那它怎么知道上一轮循环的值，从而计算出本轮循环的值？这是因为 JavaScript 引擎内部会记住上一轮循环的值，初始化本轮的变量 i 时，就在上一轮循环的基础上进行计算，厉害吧

6. 花絮

1
2
3
4
5
6

var a = [];
for (let i = 0; i < 3; i++) {
  let i = "abc";
  console.log(i);
}
// 3 abc

是不是觉得好奇怪，为啥在循环块里重复声明没有报错，因为不是在同一个块里的呀，所以就不算重复声明

设置循环变量的那部分是一个父作用域，而循环体内部是一个单独的子作用域 所以没报错，而且子作用域把父作用域的变量 i 覆盖了，所以输出了三个 abc

我本来想看看覆盖前 i 的值

1
2
3
4
5
6
7

var a = [];
for (let i = 0; i < 3; i++) {
  console.log(i);
  let i = "abc";
  console.log(i);
}
// ReferenceError: Cannot access uninitialized variable.

结果报错，忘了 let 声明的变量不能提前使用，于是我又改了改

1
2
3
4
5
6
7
8

var a = [];
for (let i = 0; i < 3; i++) {
  console.log(i);
  i = "abc";
  console.log(i);
}
// 0
// abc

这次不报错了
