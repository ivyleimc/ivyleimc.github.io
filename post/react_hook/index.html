为什么有 Hook？

回顾 Class 组件，函数组件

Class 组件

  1. 有生命周期，有实例，有 state，setState，一旦大了就复杂难拆分
  2. 相同业务逻辑，分散到各个方法中
    * 获取数据：加载完要执行，更新完也可能要执行，要写在不同的生命周期函数里面
    * 定时器：加载完设置定时器，卸载要清除定时器，要写在不同的生命周期函数里面
  3. 复用逻辑变的复杂，HOC/Render Prop

函数组件

  1. 没有生命周期，没有实例，没有 state， 没有 setState，传人 props，返回 jsx
  2. 是一个纯函数，执行完就销毁， 不保存 state
  3. 太简单

Class 组件太复杂，函数组件太简单，所以 Hook 就来了

Hook 简介

模拟了 class 组件的 state 和 setState（useState） 模拟了 class 组件的 生命周期（useEffect）
